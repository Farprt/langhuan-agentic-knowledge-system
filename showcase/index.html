LOCAL-FIRST · OBSIDIAN-NATIVE · AGENT-READY

让 Agent 使用你的知识库，而不是淹没在上下文里。

Langhuan 是面向 Obsidian / Markdown 项目的本地优先知识基础设施：结构化 RAG、增量同步、可选长期记忆，以及可替换的可观测出口。

5 分钟了解项目 查看参考知识库

配套但不绑定：公开知识库是参考实现，任意相近目录结构都可通过一个 profile 接入。

SYSTEM BOUNDARY

三个稳定契约，避免绑定某个 Agent 或云平台

项目配置决定知识边界，CLI / JSON 负责 Agent 调用，本地事件模型隔离可观测服务。

01 / PROFILE

知识接口

一个 langhuan.toml 描述目录、frontmatter、检索 scope 和非敏感运行选项。

  • Obsidian 结构化解析
  • 项目级检索范围
  • 路径与配置可移植

02 / CONTEXT

Agent 接口

Codex、Hermes 或其他 Agent 通过命令行获取带来源的上下文，并在正式编辑后显式同步。

  • 稳定 CLI / JSON
  • 来源与 heading 可核验
  • Agent 失败不损坏知识库

03 / EVENTS

运行接口

JSONL 是本地运行事实源；Honcho、AgentLoop 和 Langfuse 都位于可选集成层。

  • 云端故障软失败
  • 正文采集默认关闭
  • 集成逻辑与检索分离

ENGINEERING EVIDENCE

展示可验证的工程闭环，不堆功能名词

生产案例已在数千级 chunk 的私人知识库中运行；开源 v0.1 的能力与结论只以本仓 demo-vault 和测试为准。

Parser

理解 Obsidian，而不只是读取纯文本

frontmatter、wikilink、embed、标题层级和来源上下文进入统一 chunk metadata。

Retrieval

混合召回与重排分离

Dense 与 BM25 经 RRF 融合，再由可选本地 Cross-Encoder 重排；默认 Hash 后端只用于离线演示和接口验证。

Reliability

增量更新也有安全边界

公开 v0.1 提供文件哈希、删除检测、索引原子替换与 chunk ID 审计；跨进程锁和批次恢复列入后续路线图。

PRIVACY BY DEFAULT

本地优先不是口号，而是故障边界

DEFAULT

始终留在本地

原始 Markdown、本地索引、模型缓存、JSONL 事件和所有凭据。

OPT-IN

可选外部能力

Honcho 稳定记忆、AgentLoop / Langfuse Trace。正文和本机路径默认不进入导出载荷。

NEVER

不进入 Git

私有知识、真实日志、数据库、pickle、DPAPI 密文、租户标识、API Key 与供应商二进制。

BUILD VS. INTEGRATE

明确哪些是自研，哪些是系统集成

BUILT

Obsidian parser、heading-aware chunking、混合检索编排、增量同步、一致性审计、事件模型与安全边界。

ADAPTER BOUNDARY

SentenceTransformers / BGE 与 Cross-Encoder 可从本地路径加载；Honcho、AgentLoop 与 Langfuse 目前只提供去敏边界和配置模板。
